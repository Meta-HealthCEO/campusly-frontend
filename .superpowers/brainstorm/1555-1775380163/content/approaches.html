Three Approaches for Math Diagram Rendering

All use server-side rendering. The difference is what the AI generates and how it gets rendered.

A

Full TikZ Pipeline

Claude generates TikZ/LaTeX code for every diagram → backend renders via TeX Live (Docker) → returns clean SVG → frontend displays it.

Claude → TikZ code → TeX Live (Docker) → SVG → stored as URL → <img>

Pros

  • TikZ is the gold standard for academic math diagrams
  • Claude is exceptionally good at generating TikZ
  • Handles ALL math diagram types (geometry, graphs, constructions, Venn, number lines, 3D)
  • Output is publication-quality — identical to what you'd see in a CAPS textbook
  • Massive TikZ ecosystem (pgfplots for graphs, tkz-euclide for geometry, circuitikz for science later)

Cons

  • TeX Live Docker image is large (~1-2GB)
  • Rendering takes 1-3 seconds per diagram
  • TikZ errors can be cryptic to debug
  • Adding interactivity later requires mapping SVG elements back to data
B

Structured JSON + Node Canvas Renderers

Claude generates structured JSON describing the diagram → backend renders with Node.js libraries (D3, sharp, canvas) → returns SVG/PNG.

Claude → JSON spec → Custom Node renderers → SVG → stored as URL → <img>

Pros

  • No Docker/LaTeX dependency — runs in your existing Node.js backend
  • Structured data makes interactivity much easier later (JSON → React component)
  • Faster rendering (milliseconds, not seconds)
  • Full control over styling and output

Cons

  • Must build a renderer for every diagram type (graphs, geometry, Venn, number lines, etc.)
  • Months of work to match TikZ quality across all CAPS topics
  • Claude's JSON descriptions may lose nuance that TikZ captures naturally
  • Risk of inconsistent quality across diagram types
C

TikZ Primary + Structured Data Sidecar ★ RECOMMENDED

Claude generates both TikZ code AND a structured JSON description for each diagram. Backend renders TikZ → SVG for display now. JSON is stored alongside for future interactivity.

Claude → { tikz: "...", data: {...} } → TeX Live → SVG (display)
                                         → JSON stored (future interactive use)

Pros

  • Publication-quality diagrams now via TikZ
  • Structured data ready for interactive renderers later
  • Best of both worlds — no quality compromise, no future rework
  • JSON sidecar enables: search by diagram properties, accessibility descriptions, difficulty analysis
  • Can progressively replace TikZ with interactive renderers per diagram type

Cons

  • AI prompt is more complex (must output both formats)
  • More tokens per question generation (cost ~20-30% higher)
  • Still needs TeX Live Docker for Phase 1
