Create a Payment

Initiate a payment through any supported channel. One Gate automatically routes to the optimal processor and returns a unified response regardless of payment method.

POST /v1/payments

Request Body

ParameterTypeDescription
amount * integer Amount in cents. e.g. 10000 = R 100.00
currency * string ISO 4217 code. Default: ZAR
channel * string card eft google_pay apple_pay crypto
reference string Your internal reference for this payment
metadata object Key-value pairs for your own use

Response

200 Payment created successfully

Developer Portal — Key Features

  • Three-panel layout — nav sidebar, doc content, live code examples (like Stripe docs)
  • Multi-language code samples — cURL, Node.js, Python, PHP, Ruby with one-click copy
  • Interactive sandbox — test API calls with sandbox keys directly from docs
  • API Reference — auto-generated from OpenAPI spec, always in sync
  • Webhook events — full event catalog with signature verification docs
  • SDKs — official libraries for Node.js, Python, PHP, Ruby
  • Changelog — versioned API changes, deprecation notices
  • Developer dashboard — manage API keys, view request logs, set webhook URLs
  • OAuth 2.0 — for third-party integrations connecting on behalf of merchants

API Surface — Full Scope

Payments

  • POST /v1/payments — Create payment
  • GET /v1/payments/:id — Get status
  • POST /v1/payments/:id/refund — Refund
  • POST /v1/payments/:id/capture — Capture auth
  • GET /v1/payments — List payments
  • POST /v1/subscriptions — Recurring

Commerce

  • CRUD /v1/products — Product catalog
  • CRUD /v1/inventory — Stock management
  • CRUD /v1/orders — Order management
  • CRUD /v1/customers — Customer records
  • GET /v1/customers/:id/history — Purchase history

Analytics

  • GET /v1/reports/revenue — Revenue reports
  • GET /v1/reports/reconciliation — Recon status
  • GET /v1/insights — AI recommendations
  • GET /v1/forecasts/cashflow — Cash flow predictions
  • GET /v1/forecasts/demand — Demand predictions

Integrations

  • POST /v1/integrations/xero/sync — Sync to Xero
  • POST /v1/integrations/sage/sync — Sync to Sage
  • GET /v1/webhooks — List webhooks
  • POST /v1/webhooks — Register webhook
  • GET /v1/webhooks/events — Event types
