# 30 — Teacher Workbench (TeacherOS)

## 1. Module Overview

The Teacher Workbench is a standalone bolt-on module (`teacher_workbench`) that automates the most time-consuming parts of a teacher's daily work. It provides seven integrated tools: curriculum mapping, question banking, paper building with memos, HOD moderation, unified marking, term assessment planning, and holistic student views.

### Curriculum Engine

Teachers map their subjects to curriculum frameworks (CAPS, IEB, Cambridge, or custom). Each framework contains a hierarchical topic tree scoped by subject and grade level (1-12), organised by term (1-4). Teachers track coverage by marking topics as not_started / in_progress / completed / skipped, with optional notes and lesson plan links. A coverage dashboard shows progress bars per term with pace indicators.

### Question Bank

Teachers build a reusable question library. Each question has rich metadata: subject, grade, topic (linked to curriculum), question type (MCQ, structured, essay, true/false, matching, short answer, fill-in-blank), marks, difficulty (easy/medium/hard), cognitive level (Bloom's taxonomy: knowledge through evaluation), model answer, marking notes, images (uploaded via the image endpoint), and tags. Questions can be created manually, generated by AI, or imported from existing generated papers. Usage tracking records how often each question appears in papers.

### Paper Builder

A 3-column layout (question bank browser | paper preview | config panel) where teachers assemble exam papers by selecting questions from their bank. The paper is organised into sections. Running totals show marks, question count, and difficulty/cognitive level distributions. Teachers can use "AI Fill Gaps" to auto-generate questions for uncovered topics. Papers can be saved as drafts and exported as PDF via the browser print dialog.

### Memo Generator

When a paper is created, teachers generate a marking guide (memo) via AI. The AI produces model answers, mark allocation breakdowns (criteria that sum to the question's total marks), common mistakes (2-3 per question), and acceptable alternative answers. Teachers can edit any part of the memo and regenerate individual answers. Memos are exported as PDF alongside the paper. Mathematical equations in questions and answers are auto-detected and rendered using KaTeX.

### Paper Moderation

Single-level moderation: teachers submit papers to their HOD (determined by matching the teacher's department to the subject department). HODs review the paper and memo, then approve or request changes with comments. A moderation history array provides an audit trail. School admins can moderate any paper.

### Marking Hub

A unified inbox aggregating all pending marking from three sources: homework submissions (ungraded), assessment marks (missing entries), and AI grading jobs (needing teacher review). Items are prioritised by due date (overdue = high, due today = medium, upcoming = low). Clicking an item navigates to the appropriate existing marking page.

### Term Assessment Planner

A calendar-based view where teachers plan assessments for a term. Each planned assessment has a type (test/exam/assignment/practical/project), date, marks, weight (percentage of term mark), and linked curriculum topics. The planner provides clash detection (warns when multiple assessments are on the same day for a class), weighting validation (shows formal vs required weight percentages), and topic coverage indicators. Planned assessments can be promoted to actual assessments in the academic module.

### Student 360

A holistic view of any student aggregating data from five domains: academic performance (marks, trends, subject breakdown), attendance (rate, patterns), behaviour (discipline incidents, merit/demerit balance), homework (submission rate, late/missing counts), and parent communication (last contact date, message count). Accessed from class rosters, gradebook, or marking hub via "View Student" links.

### Roles and Access

All endpoints require authentication. Most are accessible to `teacher`, `school_admin`, and `super_admin`. Coverage updates and marking hub are teacher-only. Moderation submission is teacher-only; moderation review requires school_admin or matching department HOD.

## 2. Backend API Endpoints

All endpoints mounted under `/api/teacher-workbench`. Guarded by `requireModule('teacher_workbench')`.

---

### Dashboard

#### GET /api/teacher-workbench/dashboard

Returns aggregated stats and recent activity for the teacher's workbench.

**Auth**: teacher, school_admin, super_admin

**Response**: `{ coveragePercentage, questionCount, pendingModeration, markingItemsDue, recentActivity[] }`

---

### Curriculum Frameworks

#### GET /api/teacher-workbench/curriculum/frameworks

List all curriculum frameworks for the school.

**Auth**: teacher, school_admin, super_admin
**Query**: `schoolId`

#### POST /api/teacher-workbench/curriculum/frameworks

Create a new curriculum framework.

**Auth**: teacher, school_admin, super_admin
**Body**: `{ schoolId, name, description?, isDefault? }`

---

### Curriculum Topics

#### GET /api/teacher-workbench/curriculum/topics

List topics with filters.

**Auth**: teacher, school_admin, super_admin
**Query**: `schoolId, frameworkId?, subjectId?, gradeLevel?, term?`

#### POST /api/teacher-workbench/curriculum/topics

Create a single topic.

**Auth**: teacher, school_admin, super_admin
**Body**: `{ schoolId, frameworkId, subjectId, gradeLevel, parentTopicId?, name, description?, term, orderIndex?, cognitiveLevel?, estimatedPeriods? }`

#### PUT /api/teacher-workbench/curriculum/topics/:id

Update a topic.

**Auth**: teacher, school_admin, super_admin
**Body**: Partial topic fields (name, description, term, orderIndex, cognitiveLevel, estimatedPeriods, parentTopicId)

#### DELETE /api/teacher-workbench/curriculum/topics/:id

Soft delete a topic and its children.

**Auth**: teacher, school_admin, super_admin

#### POST /api/teacher-workbench/curriculum/topics/bulk

Bulk import topics from a JSON array (max 200).

**Auth**: teacher, school_admin, super_admin
**Body**: `{ schoolId, frameworkId, subjectId, gradeLevel, topics[] }`

---

### Curriculum Coverage

#### GET /api/teacher-workbench/curriculum/coverage

Get coverage records for a teacher's class/subject.

**Auth**: teacher, school_admin, super_admin
**Query**: `schoolId, teacherId?, classId?, subjectId?`

#### PATCH /api/teacher-workbench/curriculum/coverage/:topicId

Update or create coverage status for a topic (upsert).

**Auth**: teacher, super_admin
**Body**: `{ classId, status, dateCovered?, notes?, linkedLessonPlanId? }`

#### GET /api/teacher-workbench/curriculum/coverage/report

Coverage summary percentages per term.

**Auth**: teacher, school_admin, super_admin
**Query**: `schoolId, teacherId?, subjectId?, classId?`

---

### Question Bank

#### GET /api/teacher-workbench/questions

Search and filter questions with pagination.

**Auth**: teacher, school_admin, super_admin
**Query**: `schoolId, subjectId?, gradeLevel?, topicId?, difficulty?, cognitiveLevel?, questionType?, source?, tags?, search?, page?, limit?`

#### POST /api/teacher-workbench/questions

Create a question.

**Auth**: teacher, school_admin, super_admin
**Body**: `{ schoolId, frameworkId, subjectId, gradeLevel, topicId?, questionText, questionType, marks, difficulty, cognitiveLevel, modelAnswer?, markingNotes?, images?, options?, tags?, source? }`

#### GET /api/teacher-workbench/questions/:id

Get question detail.

#### PUT /api/teacher-workbench/questions/:id

Update a question. Body: partial question fields.

#### DELETE /api/teacher-workbench/questions/:id

Soft delete a question.

#### POST /api/teacher-workbench/questions/import-from-paper/:paperId

Extract questions from a generated paper into the question bank. Infers question type, difficulty, and cognitive level from section metadata.

**Body**: `{ frameworkId }`

---

### Image Upload

#### POST /api/teacher-workbench/uploads/image

Upload a single image for question diagrams/figures.

**Auth**: teacher, school_admin, super_admin
**Content-Type**: multipart/form-data
**Field**: `image` (JPEG, PNG, GIF, WebP; max 5MB)
**Response**: `{ url: '/uploads/teacher-workbench/uuid.ext' }`

---

### Paper Memo

#### POST /api/teacher-workbench/memos/generate/:paperId

AI generates a complete memo (marking guide) for a paper. Creates model answers, mark allocations, common mistakes, and acceptable alternatives for every question.

**Auth**: teacher, school_admin, super_admin

#### GET /api/teacher-workbench/memos/paper/:paperId

Get the memo for a paper.

#### PUT /api/teacher-workbench/memos/:id

Edit memo sections and/or status (draft/final).

**Body**: `{ sections?, status? }`

---

### Paper Moderation

#### POST /api/teacher-workbench/moderation/submit/:paperId

Teacher submits a paper for HOD review. Creates a moderation record with status `pending`.

**Auth**: teacher, super_admin

#### GET /api/teacher-workbench/moderation/queue

Get pending papers for moderation (HOD/admin view).

**Auth**: teacher, school_admin, super_admin

#### GET /api/teacher-workbench/moderation/:paperId

Get moderation status for a specific paper.

#### POST /api/teacher-workbench/moderation/:paperId/review

HOD approves or requests changes on a paper.

**Body**: `{ status: 'approved' | 'changes_requested', comments? }`

---

### Marking Hub

#### GET /api/teacher-workbench/marking-hub/pending

Aggregated pending marking items across homework, assessments, and AI grading jobs.

**Auth**: teacher, super_admin

---

### Term Planner

#### GET /api/teacher-workbench/planner/:classId/:term/:year

Get the assessment plan for a class/term/year.

#### POST /api/teacher-workbench/planner

Create or update an assessment plan (upsert by schoolId+classId+subjectId+term+year).

**Body**: `{ schoolId, subjectId, classId, term, year, plannedAssessments[] }`

#### GET /api/teacher-workbench/planner/clashes/:classId/:date

Check for assessment date clashes across subjects.

#### GET /api/teacher-workbench/planner/weightings/:subjectId

Get required vs actual assessment weightings.

---

### Student 360

#### GET /api/teacher-workbench/student-360/:studentId

Aggregated student view across academic, attendance, behaviour, homework, and communication domains.

**Auth**: teacher, school_admin, super_admin

## 3. Data Models

### CurriculumFramework
`{ schoolId, name, description, isDefault, createdBy, isDeleted, timestamps }`

### CurriculumTopic
`{ schoolId, frameworkId, subjectId, gradeLevel, parentTopicId?, name, description, term, orderIndex, cognitiveLevel, estimatedPeriods, isDeleted, timestamps }`

### CurriculumCoverage
`{ schoolId, teacherId, topicId, classId, status, dateCovered?, notes, linkedLessonPlanId?, timestamps }`
Unique index: `{ teacherId, classId, topicId }`

### Question
`{ schoolId, teacherId, frameworkId, subjectId, gradeLevel, topicId?, questionText, questionType, marks, difficulty, cognitiveLevel, modelAnswer, markingNotes, images[], options[], tags[], source, usageCount, lastUsedDate?, isDeleted, timestamps }`

### PaperMemo
`{ paperId (unique), schoolId, teacherId, sections[{ sectionTitle, answers[{ questionNumber, expectedAnswer, markAllocation[{ criterion, marks }], commonMistakes[], acceptableAlternatives[] }] }], totalMarks, status, timestamps }`

### PaperModeration
`{ paperId (unique), schoolId, submittedBy, submittedAt, moderatorId?, moderatedAt?, status, comments, moderationHistory[{ status, comment, moderatorId, date }], isDeleted, timestamps }`

### AssessmentPlan
`{ schoolId, teacherId, subjectId, classId, term, year, plannedAssessments[{ title, type, plannedDate, marks, weight, topicIds[], assessmentId?, status }], timestamps }`

## 5. Acceptance Criteria

- [ ] Teachers can create curriculum frameworks (CAPS, IEB, Cambridge, custom)
- [ ] Teachers can build topic trees with hierarchy (parent/child topics)
- [ ] Teachers can track coverage per topic per class with status and notes
- [ ] Coverage dashboard shows % progress per term with pace indicator
- [ ] Teachers can create questions with full metadata (type, difficulty, cognitive level, marks, model answer, images, tags)
- [ ] Questions can be imported from existing AI-generated papers
- [ ] Question bank supports search and multi-dimensional filtering
- [ ] Paper builder assembles papers from bank questions in sections
- [ ] AI generates complete memos with model answers, mark allocations, common mistakes
- [ ] Individual memo answers can be regenerated via AI
- [ ] Mathematical equations auto-render via KaTeX in previews
- [ ] Papers and memos export as PDF via browser print
- [ ] Teachers can submit papers for HOD moderation
- [ ] HODs can approve or request changes with comments
- [ ] Moderation history provides audit trail
- [ ] Marking hub aggregates pending work from homework, assessments, and AI grading
- [ ] Term planner shows assessments on a monthly calendar
- [ ] Clash detection warns on overlapping assessment dates
- [ ] Weighting validation shows formal vs required percentages
- [ ] Student 360 shows academic + attendance + behaviour + homework + communication
- [ ] All pages have loading spinners and empty states
- [ ] All pages are mobile-responsive
- [ ] No apiClient imports in any page or component file
- [ ] All files under 350 lines
- [ ] No `any` types in workbench code
- [ ] Image upload supports JPEG, PNG, GIF, WebP up to 5MB
